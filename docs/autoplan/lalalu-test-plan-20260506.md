## Test Diagram

### New UX Flows

| Flow | Test Type | Happy Path | Failure Path | Edge Case |
|---|---|---|---|---|

| NOWPayments payment | Integration + E2E mocked IPN | Confirmed payment credits ledger | Underpaid/expired/duplicate IPN | Confirming for long period |
| Promo code redemption | Unit + integration + E2E | Registered user applies code and receives credits | Invalid/expired/exhausted/used code | Same user retries redemption after network loss |
| Admin promo generation | Unit + E2E admin | Admin creates campaign/code and sees redemption stats | Non-admin denied | Code with per-user/domain restrictions |
| Generation status | Integration + E2E | Two MP3 variants ready | Provider failure shows support id | One variant ready, second failed |

### New Codepaths

- Package selection and payment provider branch: Stripe vs NOWPayments.
- Credit source branch: promo code vs manual admin grant vs Stripe vs NOWPayments.
- Admin authorization branch: admin vs normal user vs unauthenticated.
- Credit ledger branch: purchase credit, reserve, capture, release, refund/adjustment.
- Generation branch: song vs instrumental.
- Provider branch: KIE primary vs Aporto fallback.
- Variant branch: both ready, one ready, none ready.
- Storage branch: copy success, copy retry, copy terminal failure.
- Auth branch: anonymous draft, email auth, Google OAuth, claimed draft.

### New Background Jobs

| Job | Unit Tests | Integration Tests | Chaos Test |
|---|---|---|---|
| Generation worker | Claim only unlocked queued jobs | Submit provider attempt and persist task id | Worker crashes after provider submission |
| Callback processor | Valid/invalid provider payload normalization | Duplicate callbacks idempotent | Callback arrives before local attempt exists |
| S3 copy worker | Validates MIME/size/key | Expired provider URL retry path | S3 throttling or network timeout |
| Quality gate worker | Accepts valid two-track output | Marks partial success | One variant corrupt |
| Payment webhook processor | Signature + amount matching | Duplicate event no double credit | Webhook burst 100 events |
| Promo redemption service | Validates code hash, limits, expiry | Ledger row + redemption row in one transaction | Concurrent redemption of last available code |
| Stuck job reconciler | Finds stale `locked_at` jobs | Marks needs_support after SLA | Provider API unavailable |

### New Integrations

- KIE Gemini 2.5 Pro: schema output, timeout, malformed JSON, rate limit.
- KIE Suno: task creation, callback, status polling if needed, failed generation, media URL expiry.
- Aporto fallback: adapter compatibility contract tests.
- Stripe Checkout/webhooks: session creation, signature verification, payment success/failure.
- NOWPayments invoice/IPN: signature verification, underpaid/expired/confirming/finished states.
- Supabase Auth/Postgres/RLS: user row access, service role isolation.
- AWS S3: private bucket, presigned download, copy validation.
- Sentry/logging: support id and trace id present on user-facing failures.
- Admin cabinet: role/claim enforcement and audit events.

## Friday 2am Confidence Tests

- A user buys one song through Stripe, receives exactly one credit, double webhook does not double-credit.
- A user buys through NOWPayments, IPN is duplicated and out of order, ledger still credits once.
- Two browser tabs click "Сгенерировать песню" at the same time; only one generation reserves the credit.
- KIE accepts a task, worker crashes, callback arrives, system resumes without duplicate provider task.
- KIE returns two files, first copies to S3, second copy fails; user sees one playable variant and a retry/support state.
- Provider temporary URL expires before copy; worker retries or marks needs_support with full context.
- User A cannot read User B's draft, song, payment, S3 key, or share management endpoint.
- Non-admin cannot create promo codes, adjust balances, inspect users, or view provider payloads.
- Promo code redemption under concurrent requests creates exactly one credit grant.
- Public reveal link can play only linked tracks and cannot expose owner account data.
- Prompt injection asks assistant to change balance/payment/generation state; model output is ignored for system state.
- Load test simulates 10x expected webhook/job polling traffic without DB lock pileup.
